§ 03

Perles neurodiversité — la Ritaline numérique

Pour les parents, aidants, neuros, et tous ceux qui ont déjà eu 47 onglets ouverts simultanément.

PERLE #10 File FI

Katy, en surcharge cognitive, décrit son état mental.

Katy :

"je suis une liste FI tout court"

FIFO = First In, First Out — le principe de la file d'attente. Premier entré, premier sorti. FI = juste First In. Rien ne sort. Le cerveau enregistre tout, ne libère rien. La file grandit. Infiniment. C'est une description cliniquement précise du TDAH en surcharge cognitive.

Quand une ingénieure SCADA décrit son TDAH en termes de structure de données, c'est soit du génie, soit de l'épuisement. Probablement les deux.

PERLE #11 La mouche

Katy explique pourquoi le travail itératif avec l'IA est particulièrement adapté à son TDAH.

Katy :

"il suffit que je vois une mouche voler, pour me distraire... LOL"

La mouche hypothétique est devenue un argument officiel de politique RH pour justifier l'abonnement Claude Pro remboursé par l'employeur. E=HK².

Du TDAH à la politique sociale en une métaphore entomologique.

PERLE #12 return); — version moldu

Claude livre un fichier de code. 162 erreurs.

return);   // ← le chef-d'œuvre
En programmation, return dit à l'ordinateur "j'ai fini". La parenthèse ) ferme... quelque chose d'autre. Ensemble, ils ne forment rien de légal. C'est comme terminer une phrase avec un point d'interrogation exclamatif inversé. L'ordinateur ne sait pas quoi faire. Il plante.

Même les IA font des fautes de frappe. Sauf que les leurs font planter 162 fichiers d'un coup.

→ Version technique complète en section Développeur.

Perles éditoriales — Claude, ChatGPT, et le pipeline de l'enfer

Pour les lecteurs Hors Kadre. Et pour ceux qui ont déjà géré deux outils qui ne se parlent pas.

PERLE #17 La CI éditoriale
Katy :

"Claude rédige → ChatGPT audite → Claude affine → ChatGPT fusionne → je te raconte pas la CI LOL"

Une CI — Continuous Integration — c'est un système automatisé qui vérifie le code à chaque étape. Katy a construit l'équivalent éditorial. Deux IA qui se relisent mutuellement. Orchestrées par une humaine. Aucune des deux ne sait que l'autre existe.

Pipeline éditorial digne d'une vraie équipe DevOps. Zapier en cours d'installation.

PERLE #24 return); — version technique

162 erreurs. Eclipse. La ligne coupable :

if (AppConfig.getTacheAcquis() != null) {
    return);   // ← le chef-d'œuvre
}

return termine une méthode void. ) ferme une expression. Ensemble : erreur de syntaxe inédite. Le compilateur Java, habitué à tout, n'avait pas prévu ça. Claude a inventé une nouvelle syntaxe. Le compilateur n'a pas apprécié l'innovation.

§ 06

Ce que ça dit vraiment — angle mort

Toutes ces perles viennent du même endroit. Une IA utilisée intensément, sur des semaines, pour des sujets très différents.

Ce qu'on observe n'est pas ce qu'on attendait.

L'IA devient son interlocuteur.

Claude est devenu TDAH, féminin, ironique — parce que son interlocutrice l'est.
ChatGPT reste pragmatique, structuré, froid — parce qu'on lui demande principalement de structurer.
Suno génère exactement ce qu'on lui colle — même si c'est un prompt de description.

L'IA n'a pas de personnalité propre.
Elle a la vôtre — amplifiée.

La vraie question n'est pas : "l'IA se trompe-t-elle ?"

Elle se trompe. Comme tout le monde.

"Qu'est-ce que cette IA fait à ma manière de penser ?"

Et surtout — est-ce que je choisis encore ?
Ou est-ce que je m'adapte ?

Je dis ça, je dis rien. — E=HK² 💥
